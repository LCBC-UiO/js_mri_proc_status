MRI processing status

Getting the data and process information

To use the API from command line, you will need to use curl. You need to replace the url-part containing http://localhost:6565 to the url where the webui is running. These two first commands will return the json content for the data and the processing steps. 


curl http://localhost:6565/cgi/get_data.cgi
curl http://localhost:6565/cgi/get_process.cgi

Updating data

Send request

The curl command flag -G indicates that you want to sent key-value pairs to the url as a query string. Every key-valye pair is initiated with a -d flag, and is followed by 'key=value'. The two first key-value pairs in the example are necessary to connect the data to the correct id and session. After these two, you can add as many key-value pairs as you want to update the entry. The data will always be sorted accoring to subject id and session alphabetically. 


curl -G \
    -d 'id=sub-0022' \
    -d 'ses=ses-02Skyra' \
    -d 'raw_to_sorted=running'  \
    http://localhost:6565/cgi/update_data.cgi

Request response

Status: 500

The request errored and nothing has happened.

Status: 201

The request has been successfully completed, data is added. A preview of the session that has been edited / added is returned. 

{
  "sub-0022": {
    "ses-01Skyra": {
      "raw_to_sorted": "yes",
      "sorted_to_bids": "yes",
      "t1_rating": 3,
      "fs_recon_ok": "yes"
    }
  }
}
Status: 203

The request was received, but a key-value pair has incorrectly spelled of specified key name. A json with the erroring keys is returned. 

  ["raw_t_sorted", "t1_rat"]

Updating the process

Send request

To update the process is more or less the same type of command, just to another end-point. There is currently no way to re-over the processing steps other than manual intervetion on the file directly. 


curl -G \
  -d 'process_1=options'  \
  -d 'process_2=numeric'  \
  http://localhost:6565/cgi/update_process.cgi
Key

In the case of adding new processing steps, the key should be in snake_case (spaces substituted by underscores). Make sure you are dilligent about the spelling, it will be added 'as is'.

Value

The value of a new process may be either numeric, icons, asis (string variables, like comments), or a custom array of options (ex. ["regular", "iregular"]). All icon options have the same three valid entries: ["yes", "no", "running"], any other value is displayed in the UI as i.e. (unknown).

Status: 201

The request was received, and the file successfully updated. Returns the current state of the file. 

  ["raw_t_sorted", "t1_rat"]
Status: 203

The request was received, but a key has been provided that already exists. Returns the value of the key(s) that already exists. Nothing is updated. 

  ["raw_to_sorted", "t1_rating"]
Status: 204

The request was received, but a value-option was provided that is not accepted. Returns the vale-option that is rejected. Nothing is updated. 

{
  "icon_key": "string comment"
}

The process

The valid fields for the process are specified in their own file. This way, we can ensure that the fields for all entries are correctly spelled and displayed in the same order. Adding another field to the process is a matter of contacting the API to add a new field. This can be done here in the webui, or from the command line with curl.
